# Chronoscope

> 一台"观时之镜"——在同一条时间轴上同时容纳「138 亿年前」和「今天下午」。

Chronoscope 是一个记录事件的 web app，但不是普通记事本。它的核心是一条**跨尺度时间轴**：从地质 / 宇宙级别的大事件，到某个下午发生的小事件，都能落在同一条轴上对比先后与跨度。

## 特点

- **跨 ~12 个数量级的时间**：自制的"十进制年 + 精度档位"时间模型，支持公元前、地质年代，下至秒级。
- **模糊与残缺时间**：可记录"约 38 亿年前""1990 年代"这类模糊时间，以及只有起点或只有终点的事件。
- **双轴可视化**：上方对数总览一屏看全所有尺度，拖动"取景窗"在下方线性细节轴精细比对。
- **列表视图**：与时间轴共享筛选状态，方便精确比对。
- **可嵌套分类 + 多标签**：按分类树或标签筛选；全文搜索；事件间可关联。
- **本地优先、数据自有**：纯客户端 PWA，无后端、无账号、完全离线可用；通过导出 / 导入文件在设备间搬运，**导入即合并**不丢数据。
- **昼夜双主题**：精密仪器 / 历表气质，颜色随时间深度变温。

## 技术栈

React + TypeScript + Vite · IndexedDB（Dexie）· PWA · Canvas + DOM 混合渲染时间轴。

## 本地开发（脚手架就绪后）

```bash
npm install
npm run dev     # 启动开发服务器
npm test        # 运行单元测试（Vitest）
```

## 架构原则

纯客户端、本地优先；模块高度解耦：领域层（时间 / 模型）零依赖且纯函数，存储藏在仓储接口之后，视图是数据与筛选状态的平等消费者，功能模块可独立增减。详见设计文档。
